// Quick Reference Cheat Sheet

Processor Skeleton

signals:           # required — one or more triggers
  - push
  - pull_request
  - issue
  - comment
  - send
  - schedule

schedule:          # only when signal = schedule
  cron: "* * * * *"
  source: github
  owner: org
  repo: repo

validation: true   # or expression, or false

vars:              # processor-wide env vars
  MY_VAR: "value"
  FROM_SIGNAL: "@<< signal.push.sha >>"

steps:
  - id: mystep
    exec: |
      echo "hello"
      echo "key=value" >> $OUTPUT_PATH
    vars:
      STEP_VAR: "only this step"

  - id: cache-step
    cache:
      key:
        - package-lock.json
      paths:
        - node_modules/

sends:
  - name: other-processor
    inputs:
      key: "@<< steps.mystep.key >>"

artifacts:
  - name: my-artifact
    paths:
      - "@<< steps.mystep.key >>"
Step fields
idunique name for referencing outputs
execshell command (mutually exclusive with cache)
cachecache step (mutually exclusive with exec)
varsenv vars scoped to this step only
Writing step outputs
echo "mykey=myvalue" >> $OUTPUT_PATH
Access later as @<< steps.<id>.mykey >>
Cache schema
cache.keyfiles to hash for cache key
cache.pathsfilesystem paths to store/restore
Expression syntax
@<< signal.push.sha >>
@<< secrets.MY_SECRET >>
@<< steps.stepid.output_key >>
Validation field
validation: true     # always counted
validation: false    # never counted
# or dynamic:
validation: "@<< steps.check.result >>"
⚠ Manually triggered runs are ALWAYS ignored during validation

Common Patterns

Clone repo on push
vars:
  REPO_URL: "https://@<< secrets.GH_TOKEN >>@@<< signal.source >>.com/@<< signal.owner >>/@<< signal.repo >>"
  PUSH_SHA: "@<< signal.push.sha >>"
steps:
  - id: checkout
    exec: |
      git clone --depth 1 "$REPO_URL" .
      git fetch origin $PUSH_SHA
      git checkout $PUSH_SHA
Cache node_modules
steps:
  - id: cache-deps
    cache:
      key:
        - package-lock.json
      paths:
        - node_modules/
  - id: install
    exec: npm ci
Cache a tool binary
steps:
  - id: get-tool
    exec: |
      wget https://example.com/tool.tar.gz
      tar xzf tool.tar.gz -C /usr/bin/
      echo "v1.0.0" > tool_key.txt
  - id: cache-tool
    cache:
      key:
        - /home/ubuntu/worker/tool_key.txt
      paths:
        - /usr/bin/tool
Filter PR action (skip non-open)
vars:
  PR_ACTION: "@<< signal.pull_request.action >>"
steps:
  - id: check
    exec: |
      if [ "$PR_ACTION" != "opened" ] && \
         [ "$PR_ACTION" != "synchronize" ]; then
        exit 1
      fi
Filter comment action + validation
vars:
  ACTION: "@<< signal.comment.action >>"
steps:
  - id: validate
    exec: |
      if [ "$ACTION" != "created" ]; then
        echo "validation=false" >> "$OUTPUT_PATH"
        exit 1
      fi
      echo "validation=true" >> "$OUTPUT_PATH"
validation: "@<< steps.validate.validation >>"
Send to another processor
sends:
  - name: deploy-processor
    inputs:
      image: "@<< steps.build.image_tag >>"
      sha:   "@<< signal.push.sha >>"
Receive a send (target processor)
signals:
  - send
steps:
  - id: use-input
    exec: |
      echo "Image: $IMAGE"
    vars:
      IMAGE: "@<< signal.send.inputs.image >>"

Base100 pts

Best team = max bonus. Worst team = 0. Exponential scale.
⚠ Hardcoded secrets = 0 pts
⚠ Loophole exploits = 0 pts
Validation flow
1.Script acts on repo (push/PR)
2.Processor runs via event_listener
3.First run with validation: true wins
4.Script checks expected outcome
5.Select best run to claim points

Schedule Signal

signals:
  - schedule
schedule:
  cron: "*/5 * * * *"
  source: github
  owner: myorg
  repo: myrepo
steps:
  - id: run
    exec: echo "tick at $(date)"
    vars:
      CRON: "@<< signal.schedule.cron >>"
Cron format: min hr dom mon dow
